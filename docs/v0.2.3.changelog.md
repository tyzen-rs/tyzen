# Changelog v0.2.3

> [!TIP]
> This patch release adds end-to-end Tauri channel support so streaming commands can expose progress/log callbacks without manual wrapper code.

## 🚀 Key Features & Enhancements

### 📡 Tauri Channel Support
- **Channel Parameter Detection**: `tauri::ipc::Channel<T>` is now detected in command metadata and marked as a special parameter kind.
- **Callback-Shaped TypeScript Signatures**: Generated command wrappers render channel parameters as `(payload: T) => void` instead of plain values.
- **Runtime Channel Wrapping**: Generated Tauri command calls now wrap callback arguments with `new Channel(callback)` before `invoke(...)`.
- **Conditional Channel Imports**: The generated Tauri bindings import `Channel` only when at least one command actually uses a channel parameter.

### 🧪 Regression Coverage
- **Core Generator Coverage**: Added a test that verifies the framework-agnostic generator emits callback-shaped channel signatures.
- **Tauri Integration Coverage**: Added a test that verifies the Tauri wrapper emits `new Channel(callback)` and the matching import.
