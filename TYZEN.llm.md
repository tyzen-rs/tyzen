# Tyzen LLM Contract

## Overview
Tyzen is a metadata-driven type synchronization system. It uses `inventory` to collect type and command definitions at compile-time and emits TypeScript source code at runtime.

## Core Contract

### Type Registration
**Macro**: `#[derive(tyzen::Type)]`
**Trait**: `tyzen::TsType { fn ts_name() -> String }`

**Inputs (Rust Attributes)**:
- `#[serde(rename = "...")]`: Overrides the TypeScript type name.
- `#[serde(rename_all = "...")]`: Applies case conversion to fields (snake_case, camelCase, kebab-case, lowercase, UPPERCASE, PascalCase).
- `#[serde(tag = "...", content = "...")]`: Controls enum representation (externally tagged, internally tagged, adjacently tagged).
- `#[serde(untagged)]`: Represents enums as a union of raw values.
- `#[serde(flatten)]`: Inlines fields from another struct.
- `#[tyzen(optional)]`: Marks a field as optional (`?`) in TypeScript, regardless of `Option<T>` wrapper.

### Variant Metadata
**Macro**: `#[tyzen(key = "value")]` on enum variants.

**Contract**:
1. If any variant has a `#[tyzen]` attribute, an `export const [EnumName]Meta` object is generated.
2. Keys in the metadata object match the TypeScript variant names (respecting renames).
3. Metadata keys (e.g., `error_code`) are converted to camelCase in TypeScript.
4. Variants without metadata are included as empty objects `{}`.
5. Users can override the meta object name using `#[tyzen(meta = "CustomName")]` on the enum.

**Outputs (TypeScript)**:
- Structs → `export type Name = { ... }`
- Unit Enums → `export const Name = { ... } as const` + `export type Name = (typeof Name)[keyof typeof Name]`
- Complex Enums → Discriminated Union of types.

### Command Registration (Tauri)
**Macros**: `#[tauri::command]` AND `#[tyzen_tauri::command]`

**Rules**:
1. Both attributes are mandatory for inclusion in generated bindings.
2. Return types are automatically mapped to `Promise<Result<T, E>>`.
3. If the Rust return is `T` (not `Result`), it is mapped to `{ status: 'ok', data: T }`.
4. If the Rust return is `Result<T, E>`, it is mapped to `{ status: 'ok', data: T } | { status: 'error', error: E }`.

**Input Detection**:
- `tauri::ipc::Channel<T>`: Detected automatically. The TS wrapper accepts `(payload: T) => void`.
- `State<T>`, `AppHandle`, `Window`: Filtered out from TS signatures; they are injected by Tauri at runtime.

### Generation Workflow
- `tyzen_tauri::generate(path)`: Primary entry point.
- Atomicity: Files are only overwritten if the content has changed to prevent unnecessary rebuilds.
- Header: All files start with `// auto-generated by tyzen, do not edit`.

## Edge Cases

| Condition | Behavior |
|---|---|
| Duplicate type names | TypeScript naming collision (no automatic namespacing yet). |
| Missing `TsType` impl | Compile-time error on the struct/enum using the type. |
| Nested `Option<Option<T>>` | Collapses to `T | null` in TypeScript. |
| `Vec<u8>` | Currently mapped to `number[]` (Binary optimization pending). |
| Path normalization | `generate` handles parent directory creation automatically. |

## Design Decisions

- **Two Attributes for Commands**: Prevents implicit inclusion of internal commands in the public API. Keeps the "surface area" of the bridge explicit.
- **Inventory Crate**: Used to avoid manual registration boilerplate. Allows commands to be defined anywhere in the codebase while still being discoverable by the generator.
- **CamelCase Conversion**: Automatically applied to Rust snake_case function names to follow TypeScript conventions.

- The Tauri `invoke` wrapper logic in `tyzen-tauri/src/lib.rs` changes.
